HTML SEMÂNTICO, RÓTULOS E DADOS DE FORMA ACESSÍVEL

O HTML semântico nada mais é do que, escrever o seu código de uma forma que seja de facil entendimento para o computador, ou seja, escrever o HTML com tags que façam sentido, de acordo com o conteúdo. Isso é importante para acessibilidade pois, ajuda as ferramentas como o leitor de tela, compreender melhor as seções e tudo que está disposto em seu site.

"Semântica é o estudo do significado e interpretação de palavras. Aplicando isso ao HTML, significa a melhor compreensão do HTML para o computador."

Anteriormente do atual HTML5, usavamos apenas "divs" para separar o conteudo, sendo assim o computador não podia compreender o seu site pois não havia uma separação que fazia sentido para ele, dificultando a criação e usabilidade de leitores de tela.

Temos bons exemplos de tags html semãnticas, algumas delas são: header, aside, section, article, nav, main, footer, entre outras.

Aqui vai uma breve explicação das principais tags citada acima, também utilizando uma lista ordenada que ajuda ao leitor de tela compreender a ordem e o tipo de conteúdo.

  1. Header: indica o cabeçalho da página.
  2. Section: Indica seções de conteúdo do seu site.
  3. Nav: Indica um agrupamento de links, uma barra de navegação entre as páginas e/ou o conteúdo de sua página.
  4. Main: Indica o conteúdo principal da sua página, só é recomendado um por página.
  5. Figure: Indica que o conteúdo é uma figura, e ainda é possivel adicioanru um 'figcaption', uma legenda para a mesma.
  6. Footer: Indica que o conteúdo é um rodapé, pode ser utilizado tanto na página quanto no rodapé de artigos.

Rótulos de Texto com significado

Os rótulos de texto tornam mais fácil entender para onde e para que serve o item, tais como botões e links. Devem ser feitos de maneira que o leitor entenda onde vai parar se clicar naquele local, um exemplo de um rótulo ruim são links com "clique aqui", isso pode parecer algo comum mas se deseja acessibilidade, precisa entender que, os ususarios de leitores podem usar atalhos para exibir ou ouvir listas de botões e controle de formulário, dificultando uma identificação adequada. Você deve se certificar de que os rótulos fazem sentido quando lidos individualmente e no contexto do paragráfo.

Aqui está um bom exemplo de rótulo em um link:
"Acessibilidade na web é de extrema importância para nosso mundo. veja mais sobre acessibilidade web."
Em seguida um exemplo do que não deve ser feito:
"Acessibilidade na web é de extrema importância para nosso mundo. clique aqui para saber mais."

Os rótulos de formulário também devem ser escritos de uma maneira compreensivel para os leitores, sendo assim uma boa maneira de escreve-los serie:

<div>
<label for="email">Digite o seu e-mail aqui:</label>
<input type="text" id="email" name="email">
</div>

Dessa forma o leitor entenderá que ali é um campo a ser preenchido com o e-mail.

E aqui seria um exemplo ruim a ser usado em seu código:

Digite o seu e-mail aqui: <input type="text" id="email" name="email">

MODO ESCURO

Este é o Dark Mode ou modo escuro, uma tecnologia muito importante e muito utilizada atualmente. por exemplo, ela pode ajudar pessoas que tem algum tipo de problema visual, como fotofobia, e até mesmo pessoas que não tem problemas, mas que utilizam muito o computador para leitura, trabalho e afins, pois ela diminui o esforço que os olhos precisam fazer para poder ler a tela e, a longo prazo, evita que as pessoas desenvolvam alguma doença nos olhos. Ele pode ser feito de maneira relativamente simples com CSS/Javascript. Aqui está um video explicando uma das maneiras de se fazer o Dark Mode. Esta não é a unica forma de faze-lo, mas vale dar uma olhada, pois desenvolvedores de qualquer nivel, serão capazes de aplica-la em seus projetos, ajudando as pessoas com o modo escuro em seus sites ou aplicações web.

Aqui está o código base utilizado, disponivel no video citado no texto acima. Você pode fazer as modificações de acordo com a sua necessidade! 

                    
                        const html = document.querySelector("html")
                        const checkbox = document.querySelector("input[name=theme]")
                        
                        const getStyle = (element, style) =>
                        window
                        .getComputedStyle(element)
                        .getPropertyValue(style)
                        
                        
                        const initialColors = {
                            bg: getStyle(html, "--bg"),
                            bgPanel: getStyle(html, "--bg-panel"),
                            colorHeadings: getStyle(html, "--color-headings"),
                            colorText: getStyle(html, "--color-text"),
                        }
                        
                        const darkMode = {
                            bg: "#333333",
                            bgPanel: "#434343",
                            colorHeadings: "#3664FF",
                        colorText: "#B5B5B5"
                    }
                    
                    const transformKey = key =>
                    "--" + key.replace(/([A-Z])/, "-$1").toLowerCase()
                    
                    
                    const changeColors = (colors) => {
                        Object.keys(colors).map(key =>
                        html.style.setProperty(transformKey(key), colors[key])
                        )
                    }
                    
                    
                    checkbox.addEventListener("change", ({target}) => {
                        target.checked ? changeColors(darkMode) : changeColors(initialColors)
                    })

ALTO CONTRASTE

A aplicação de alto contraste, é uma prática muito útil também, assim como o próprio dark mode, pois também facilita a leitura, dando mais foco no texto a ser lido em uma página, seu padrão, como pode ser visto, utilizando a tecla 'ALT + 3', nessa página, é transformar os fundos em preto e as fontes em branco, assim causando um grande contraste entre essas duas cores básicas, tornando muito mais legivel a quem tem alguma dificuldade para ler em telas de qualquer dispositivo, também tornando mais facil para o olho humano focar nos textos e outras coisas exibidas na tela.

Esse é o código javascript do alto contraste, feito para fazer as teclas funcionarem, no caso usamos o ALT + 1, ativando assim o alto contraste.(Lembre-se de adicionar ao seu html uma descrição explicando qual o botão deve ser pressionado para ativa-lo e também, de fazer a parte funcional direto no HTML.)

                    
                        
                        function () {
                            var Contrast = {
                                storage: 'contrastState',
                                cssClass: 'contrast',
                                currentState: null,
                                check: checkContrast,
                                getState: getContrastState,
                                setState: setContrastState,
                                toogle: toogleContrast,
                                updateView: updateViewContrast
                            };
                            
                            window.toggleContrast = function () { Contrast.toogle(); };
                            
                            Contrast.check();
                            
                            function checkContrast() {
                                this.updateView();
                            }
                            
                            function getContrastState() {
                            return localStorage.getItem(this.storage) === 'true';
                        }
                        
                        function setContrastState(state) {
                            localStorage.setItem(this.storage, '' + state);
                            this.currentState = state;
                            this.updateView();
                        }
                        
                        function updateViewContrast() {
                            var body = document.body;
                            
                            if (!body) return;
                            
                            if (this.currentState === null)
                            this.currentState = this.getState();
                            
                            if (this.currentState)
                            body.classList.add(this.cssClass);
                            else
                            body.classList.remove(this.cssClass);
                        }
                        
                        function toogleContrast() {
                            this.setState(!this.currentState);
                        }
                    })();

NAVEGAÇÃO SEM MOUSE OU VIA TECLADO

A navegação sem utilizar o mouse, é uma boa prática de acessibilidade, pois auxilia as pessoas que tem algum tipo de impedimento ou baixa mobilidade, assim, a pessoa pode utilizar apenas um dispositivo/preriférico para navegar pelo seu website. Utilizamos alguns exemplos nesta página, como no inicio da página, um link para pular para o conteúdo principal, em seguida, um link para acessar o botão de dark mode, e as teclas ALT + 3 para ativar o alto contraste, como mostrado no tópico anterior, também pode ser feita a navegação entre os tópicos da página com a tecla TAB, assim facilitando para quem utiliza leitores de tela. No nosso caso, utilizamos o tabindex no HTML, pois de forma fácil ele ajuda a organizar a ordem em que a tecla TAB navega pela página, e já aplica automáticamente a borda em volta do conteudo. Existem outras maneiras de se fazer a navegação via teclado, quando necessário para sites mais complexos e com mais elementos na página. Você pode acessar um post sobre o tabindex, do desenvolvedor Everson da Luz, clicando aqui.

BOAS PRATICAS NA DESCRIÇÃO DE CONTEÚDOS

O conteútdo em texto de um site, é inerentemente acessível porém, não devemos esquecer da acessibilidade em conteúdos multimídia como videos e imagens. Obviamente alguns não conseguem ver o conteúdo e outros não conseguem ouvir. Sendo assim, devemos aplicar regras a esse tipo de conteúdo para auxiliar na compreensão de quem necessita.

Tratando-se de imagens, se você utilizar apenas o nome do arquivo em sua composição como:

<img src="pokemon.png">

Ela não indicará nada além de nome do arquivo ao leitor de tela, isso torna dificil a compreensão pois apenas indica que é um pokemon no exemplo mas, não diz qual pokemon seria e suas caracteristícas.

<img src="pokemon.png" alt="Um pokemon chamado Mewtwo, um lendário pokemon, com aparência de felino bípede branco, alto e esguio, possui uma cauda e estômago roxos com uma espécie de segundo ducto cutâneo ligando a nuca para suas costas, dobrando o pescoço. Sua aparência é ligada a mistura de um gato, um esquilo e um canguru. Ele tem as cores cinza e roxo." title="Pokemon Lendário MewTwo" >

Já no exemplo acima, a descrição completa da imagem será exibita pelo texto alt e uma mais breve pelo atributo title.
